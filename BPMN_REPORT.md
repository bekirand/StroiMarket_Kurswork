# Детальный анализ бизнес-процесса "Оформление и обработка заказа" (BPMN)

На основе анализа вашей базы данных (`schema.prisma`), я восстановил полный цикл работы маркетплейса стройматериалов.

## Дорожки (Lanes / Pools) в BPMN:
1. **Покупатель (Customer)** — Оформляет заказ, выбирает доставку.
2. **Система (Сайт / БД)** — Фиксирует статусы `OrderStatus`, логирует историю (`OrderStatusHistory`), меняет остатки.
3. **Менеджер (Manager)** — Проверяет и подтверждает заказы.
4. **Склад (Storekeeper)** — Собирает заказ и отгружает его.

---

## Вариант 1: Быстрая автоматическая генерация в Draw.io
Вы можете скопировать следующий код, зайти в **Draw.io (diagrams.net)**, нажать **Расширенные (Arrange) -> Вставить (Insert) -> Расширенные (Advanced) -> Mermaid...** и вставить этот код. Программа сама нарисует все блоки и стрелки!

```mermaid
flowchart TD
    %% Определяем стили для блоков
    classDef startEvent fill:#d4edda,stroke:#28a745,stroke-width:2px,shape:circle;
    classDef endEvent fill:#f8d7da,stroke:#dc3545,stroke-width:4px,shape:circle;
    classDef sysAction fill:#e2e3e5,stroke:#6c757d;
    classDef gateway fill:#fff3cd,stroke:#ffc107,shape:diamond;

    %% --- ДОРОЖКА: ПОКУПАТЕЛЬ ---
    subgraph Покупатель [Участник: Покупатель (Customer)]
        Start(((Начало))) ::: startEvent --> A[Выбрать товары и добавить в корзину]
        A --> B[Оформить заказ: Выбрать тип / ADDRESS или PICKUP]
    end

    %% --- ДОРОЖКА: СИСТЕМА ---
    subgraph Система [Участник: Система (Сайт и БД)]
        B --> C[Создать Order со статусом NOVIY] ::: sysAction
        C --> D[Записать OrderStatusHistory] ::: sysAction
    end

    %% --- ДОРОЖКА: МЕНЕДЖЕР ---
    subgraph Менеджер [Участник: Менеджер (Manager)]
        D --> E{Проверить корректность заказа} ::: gateway
        E -->|Данные неверны / Нет в наличии| F[Отклонить заказ]
        E -->|Всё верно| G[Подтвердить заказ]
    end

    %% Отработка Системой решений менеджера
    F --> F_Sys[Статус: OTMENEN] ::: sysAction
    G --> G_Sys[Статус: PODTVERZHDEN] ::: sysAction
    
    F_Sys --> End1(((Конец: Отмена))) ::: endEvent
    G_Sys --> H_Sys[Записать OrderStatusHistory] ::: sysAction

    %% --- ДОРОЖКА: СКЛАД ---
    subgraph Склад [Участник: Складщик (Storekeeper)]
        H_Sys --> I[Взять заказ в работу]
        I --> I_Sys[Статус: V_OBRABOTKE] ::: sysAction
        I_Sys --> J[Собрать стройматериалы]
        J --> K[Отгрузить заказ в доставку / пункт выдачи]
    end

    %% Списание остатков
    K --> K_Sys[Списать stockQuantity товара] ::: sysAction
    K_Sys --> L_Sys[Статус: OTPRAVLEN] ::: sysAction

    %% Завершение
    L_Sys --> M[Доставить клиенту / Выдать клиенту]
    M --> N_Sys[Статус: DOSTAVLEN] ::: sysAction
    N_Sys --> End2(((Конец: Успех))) ::: endEvent
```

## Вариант 2: Детальная инструкция для ручного рисования классического BPMN
Если преподаватель требует строгую нотацию (Start Event, Task, XOR-Gateway), вот пошаговый план, как отрисовать это вручную:

### Пул: Магазин Стройматериалов (4 дорожки)
**1. Дорожка "Покупатель":**
- **Начальное событие (Зеленый круг):** "Потребность в стройматериалах"
- **Задача (User Task):** "Формирование корзины (`CartItem`)"
- **Задача (User Task):** "Оформление заказа (`Order` + `DeliveryType`)"
- *Переход в дорожку "Система"*

**2. Дорожка "Система":**
- **Задача (Service Task):** "Присвоение статуса `NOVIY` и сохранение в БД"
- *Переход в дорожку "Менеджер"*

**3. Дорожка "Менеджер":**
- **Задача (User Task):** "Рассмотрение нового заказа"
- **Шлюз (Gateway XOR ✖):** "Заказ корректен?"
  - **Ветвь "Нет":** -> Дорожка "Система" (Задача "Статус `OTMENEN`") -> **Конечное событие (Красный круг):** "Заказ отменен"
  - **Ветвь "Да":** -> Дорожка "Система" (Задача "Статус `PODTVERZHDEN`")

**4. Дорожка "Складщик":**
- **Задача (User Task):** "Принятие заказа в сборку"
- **Задача (Система):** "Статус `V_OBRABOTKE`"
- **Задача (User Task):** "Физическая сборка заказа (паллеты, мешки и т.д.)"
- **Задача (User Task):** "Передача в службу доставки"
- *Переход в дорожку "Система"*

**5. Дорожка "Система" (окончание):**
- **Задача (Service Task):** "Уменьшение остатков на складе (`stockQuantity`)"
- **Задача (Service Task):** "Смена статуса на `OTPRAVLEN`"
- *Ожидание доставки:* -> При получении клиентом -> **Задача (Service Task):** "Смена статуса на `DOSTAVLEN`"
- **Конечное событие (Красный круг):** "Заказ успешно завершен"
