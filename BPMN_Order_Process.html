Жизненный цикл статусов заказа (OrderStatus)

Новый → Подтверждён → В обработке → Отправлен → Доставлен | Отменён (при отклонении — остатки на складе восстанавливаются)

Ключевые особенности бизнес-логики (из исходного кода)

  • Авторизация обязательна для оформления: Страница /checkout редиректит неавторизованных на /login?callbackUrl=/checkout
  • Гостевая корзина: CartItem и Favorite работают для гостей через sessionId (cookie, 30 дней)
  • Списание остатков при создании: Остатки на складе уменьшаются сразу в транзакции с созданием заказа, а не при отгрузке
  • Возврат при отмене: При смене статуса на «Отменён» — остатки на складе увеличиваются обратно
  • Права менеджеров: Проверяется флаг canChangeOrderStatus из БД — admin может его отключить
  • Отзывы только после доставки: Проверяется наличие заказа со статусом «Доставлен» с данным товаром; количество отзывов ≤ количество доставленных заказов
  • Автозаполнение при оформлении: Поля имя, email заполняются из session.user, телефон — из /api/user/profile, адрес — из сохранённых Address
  • Поиск с опечатками: Реализован алгоритм Левенштейна для fuzzy-поиска по товарам
